Section 1: Data Flow & Information Sharing
Approved

The EMR facilitates the collection, use, and disclosure of Individually Identifying Health Information (IIHI) through the following provincial interfaces:

Person Directory (PD)
For identity validation (ULI/PHN). Accessed via SOAP/HL7 on every patient lookup.
Alberta Netcare
For clinical data viewing and CII/CPAR uploads. Requires physician authentication.
H-Link
For financial claim batch submissions to Alberta Health via SFTP. Fixed-length .clm format.
Alberta Precision Labs (APL)
For inbound HL7 v2.5 laboratory results. Auto-parsed into Observations table.
Section 2: Technical Security Safeguards
Implemented
Standard: Encryption at Rest and in Transit (AES-256 / TLS 1.3)
ControlImplementation DetailsStatus

Audit Logging Immutable logs track all CRUD actions for 10 years per PLAS standards. Accessible via Privacy & Audit Log module. Active
Data Residency All health information stored on Canadian-soil servers (AWS Canada Central). No cross-border data transfer. Compliant
Encryption Keys Keys managed separately from the data storage layer. Annual rotation schedule enforced. Annual Review
Database Masking Sensitive patients (clinic staff / VIPs) masked from standard queries. Requires elevated role to access. Active

In Place

Breach Notification Protocol
Automated alerts to the Custodian if an unauthorized IP attempts database access. Notification to Commissioner required within 30 days of discovery under HIA s.60.1.

Data Validation Rules (Logic Layer)
ULI
ULI Validation: Use utils/mod10.js. 10th digit must match weighted sum of first 9 digits (Alberta Mod-10 algorithm). Rejection Code 05 or 30 on failure.
DX
Diagnostic Codes: Strictly 3 or 4 character ICD-9. Do not use ICD-10 for Alberta Health Billing. Rejection Code 39EB = invalid DX code — check dx_lookup table.
DATE
Service Date: Cannot be in the future. Cannot be older than 180 days — late filing penalty applies. Format: YYYYMMDD in .clm file, no separators.
